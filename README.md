# ens-token-onchain-analysis

Комплексный On-Chain анализ 1.08 млн транзакций токена ENS (Ethereum Name Service). Проект включает в себя исследование структуры рынка, анализ концентрации владения (кривая Лоренца, индекс Джини), сетевой анализ графа переводов, кластеризацию пользователей с помощью K-Means и прогнозирование направления цены токена.

## О проекте

Этот проект представляет собой исследование блокчейн-данных токена ENS с момента его запуска (ноябрь 2021) по декабрь 2024 года. Цель работы — выявить скрытые паттерны поведения крупных держателей (китов), оценить структурную надежность сети транзакций и проверить, обладают ли On-Chain метрики предсказательной силой для прогнозирования рынка.

## Этапы работы

### 1. Исследовательский анализ и структура рынка
* Проведен анализ распределения объемов транзакций. Выявлено, что рынок подчиняется степенному распределению (Power Law) с экстремальной асимметрией (skewness = 686).
* Рассчитаны метрики концентрации богатства. Индекс Джини составил 0.9911, что указывает на критический уровень централизации (топ-5 адресов контролируют 75.4% эмиссии).

### 2. Сетевой анализ (Network Science)
* Построен направленный взвешенный граф транзакций (10.2 тыс. узлов, 23.8 тыс. ребер).
* Рассчитаны метрики центральности (PageRank, Betweenness, Degree) для выявления ключевых хабов.
* Проведен стресс-тест сети на устойчивость (Resilience Test). Доказано, что удаление всего 15 критических узлов (0.14% сети) приводит к распаду гигантской компоненты графа и падению связности сети на 48%.

### 3. Кластеризация пользователей
* Сгенерированы профили для 245 тыс. уникальных адресов на основе объема, частоты транзакций и количества контрагентов.
* Применен алгоритм KMeans на логарифмированных данных. Выделено три сегмента пользователей: Киты (11.2% адресов, контролирующих 92% объема), Аирдроп-сообщество (64.7% адресов со сверхнизкой активностью) и Ритейл (24.1% адресов).

### 4. Предиктивный анализ (Machine Learning)
* On-Chain метрики агрегированы по дням с использованием скользящего окна в 7 дней (rolling window) и сдвига (shift) для предотвращения утечки данных (data leakage).
* Задача сформулирована как бинарная классификация: предсказание направления движения цены (UP/DOWN) через 7 дней.
* Проведено сравнение моделей на кросс-валидации TimeSeriesSplit (5 фолдов).

## Результаты экспериментов

### Метрики моделей на тестовой выборке:

| Модель | Accuracy | ROC-AUC | Precision |
| :--- | :---: | :---: | :---: |
| **Logistic Regression** | **52.65%** | **0.6045** | **56.81%** |
| Neural Network (MLP) | 51.57% | 0.5722 | 55.94% |
| Random Forest | 51.24% | 0.5533 | 55.30% |
| Gradient Boosting | 49.73% | 0.5503 | 53.80% |

Простая линейная модель с L2-регуляризацией (Logistic Regression) показала наилучшую обобщающую способность на зашумленных данных. Показатель Precision = 56.81% указывает на наличие статистического преимущества (edge) при принятии торговых решений на основе On-Chain сигналов.

### Наиболее важные признаки для прогноза (Feature Importance):
1. Медианный размер транзакции за последние 7 дней (26.61%)
2. Скользящее среднее цены за 7 дней (18.17%)
3. Количество уникальных отправителей за 7 дней (12.87%)
